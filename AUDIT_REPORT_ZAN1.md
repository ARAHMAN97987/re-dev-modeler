## الباغ الحرج — تم إصلاحه

### المشكلة
`financing.js` سطور 52, 156, 575: عند حساب Exit Value، الكود يستخدم `??` (nullish coalescing) للـ fallback:
```javascript
// قبل الإصلاح:
const assetIncome = as.revenueSchedule[exitIdx] ?? as.revenueSchedule[fallbackIdx] ?? 0;
```
`??` يتجاوز `null`/`undefined` فقط — وليس `0`. إذا الإيرادات عند سنة التخارج = 0 (مثلاً التخارج عند اكتمال البناء قبل بدء التشغيل)، الـ fallback لا يعمل و exit value = 0.

### الإصلاح
```javascript
// بعد الإصلاح:
const assetIncome = as.revenueSchedule[exitIdx] || as.revenueSchedule[fallbackIdx] || 0;
```
`||` يتجاوز أي قيمة falsy بما فيها `0`. إذا الإيرادات = 0، يستخدم الـ fallback (إيرادات مستقبلية).

### التأثير
- **المشاريع الحالية**: لا تأثير (كل المشاريع تتخارج بعد بدء الإيرادات)
- **حالة حدية**: مشروع يتخارج قبل أو عند بدء التشغيل كان سيحصل على exit value = 0
